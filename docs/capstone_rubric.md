# Capstone Rubric – Self-Assessment for the Final Web Project

Use this rubric to evaluate your finished project at the end of Day 28.  
Score yourself honestly — the goal is to identify what you've learned and what to improve next.

---

## How to use this rubric

1. Read each category and its criteria.
2. Decide which score (0–3) best describes your project for that category.
3. Add up your scores (max 15 points).
4. Read the reflection questions at the bottom.

---

## Category 1 – Functionality (0–3 points)

Does the project actually work?

| Score | Description |
|-------|-------------|
| **3** | Frontend loads in the browser. All backend endpoints (`/hello`, `/info`, `/weather`) return correct responses. Frontend buttons connect to the backend and display results. |
| **2** | Most features work. At least two endpoints return correct responses. Minor issues (e.g., one button doesn't work or page styling is broken). |
| **1** | The server starts without crashing, but only one endpoint works or the frontend does not connect to the backend. |
| **0** | The server crashes on startup or no endpoint returns a correct response. |
